# CASE - 切片策略

本目录包含 6 种文本切片（Chunking）策略的独立示例脚本，以及 1 个**策略对比测试**脚本。场景以「上海迪士尼乐园门票与购票规则」为例，演示 RAG 知识库构建前如何将长文档切分为适合 Embedding 和检索的文本块。

切片质量直接影响向量检索的命中率：块太大则噪声多，块太小则语义不完整。本案例从简单到复杂，覆盖规则切片与 LLM 语义切片，便于按文档类型选型。

## 脚本说明

### 1-固定长度切片.py

**作用：** 按固定字符长度切分文本，并在切分点附近回退到句子边界，避免从句子中间截断。

**核心函数：** `improved_fixed_length_chunking(text, chunk_size=512, overlap=50)`

| 参数 | 默认值 | 说明 |
|------|--------|------|
| chunk_size | 512 | 目标块大小（字符数） |
| overlap | 50 | 相邻块重叠字符数，减少边界信息丢失 |

**切分逻辑：** 到达 `chunk_size` 位置后，向前最多 100 字符内寻找 `.!?。！？` 等句末标点，在标点处断开；下一块起点为 `end - overlap`。

**优点：** 实现简单，块大小可控，适合结构规整的纯文本。

**缺点：** 不考虑段落/标题结构，可能把主题相近但分属不同段落的内容切在一起。

---

### 2-句子边界切片.py

**作用：** 先按句子拆分，再将多个句子合并为不超过 `max_chunk_size` 的块，保持句子完整性。

**核心函数：** `semantic_chunking(text, max_chunk_size=512)`

**切分逻辑：** 用正则 `[.!?。！？\n]+` 分句，逐句累加；超过最大长度时保存当前块并开始新块。

**优点：** 语义边界清晰，不会在半句处切断。

**缺点：** 块长度不均匀；单句超长时可能产生超大块。

---

### 3-LLM语义切片.py

**作用：** 调用大模型按**语义完整性**切分文本，在自然的主题分界处断开，返回 JSON 格式的切片列表。

**核心函数：** `advanced_semantic_chunking_with_llm(text, max_chunk_size=512)`

**依赖：** 需配置 `DASHSCOPE_API_KEY`，通过 OpenAI 兼容接口调用 DashScope。

**默认模型：** `qwen-turbo-latest`（若遇 403，可改为 `qwen-plus`，与对比测试脚本一致）

**降级逻辑：** 未设置 API Key 时，脚本会尝试调用 `fallback_semantic_chunking`；若该函数未定义，需自行配置 Key 或改用脚本 2 的规则切片。

**优点：** 切分最符合人类阅读逻辑，适合主题多变的长文档。

**缺点：** 需 API 调用，有成本与延迟；返回格式依赖模型遵循 JSON 约定。

---

### 4-层次切片.py

**作用：** 根据文档**标题层次**切分，遇到一级/二级标题时优先开新块，适合 Markdown 或带章节结构的文档。

**核心函数：** `hierarchical_chunking(text, target_size=512, preserve_hierarchy=True)`

**识别的层次标记：**

| 级别 | 标记示例 |
|------|----------|
| title1 | `# `、`标题1：`、`一、`、`1. ` |
| title2 | `## `、`标题2：`、`二、`、`2. ` |
| title3 | `### `、`标题3：`、`三、`、`3. ` |
| paragraph | 空行、换行 |

**开新块条件：**

1. 遇到 `title1` / `title2` 且 `preserve_hierarchy=True`
2. 当前块长度超过 `target_size`
3. 段落级内容且当前块已达 `target_size * 0.8`

**优点：** 保留章节结构，检索结果更易对应文档出处。

**缺点：** 依赖标题格式规范；纯段落文本效果接近普通切分。

---

### 5-滑动窗口切片.py

**作用：** 固定窗口长度、固定步长的滑动切分，相邻块之间有重叠区域。

**核心函数：** `sliding_window_chunking(text, window_size=512, step_size=256)`

| 参数 | 说明 |
|------|------|
| window_size | 每个窗口的字符长度 |
| step_size | 每次滑动的步长（小于 window_size 即产生重叠） |

**优点：** 重叠保证跨边界的语义不被截断，召回率往往更高。

**缺点：** 切片数量多、存储与检索成本增加，重复内容多。

---

### 6-自适应切片.py

**作用：** 以**段落**（`\n\n` 分隔）为基本单位，在目标长度附近动态合并，允许一定容差。

**核心函数：** `adaptive_chunking(text, target_size=512, tolerance=0.2)`

**切分逻辑：** 段落逐段累加；超过 `target_size * (1 + tolerance)` 时结束当前块。`tolerance=0.2` 表示允许超出目标长度 20%。

**优点：** 兼顾段落语义与长度控制，适合新闻、说明类多段落文本。

**缺点：** 单段超长时仍可能产生大块。

---

### 切片策略对比测试.py

**作用：** 在**同一段测试文本**上依次运行多种切片策略，输出统计对比与简单推荐度评分。

**包含策略（6 种）：**

| 序号 | 策略 | 对应独立脚本 |
|------|------|--------------|
| 1 | 固定长度切片 | `1-固定长度切片.py` |
| 2 | 句子边界切片 | `2-句子边界切片.py` |
| 3 | LLM 语义切片 | `3-LLM语义切片.py`（本脚本使用 `qwen-plus`） |
| 4 | 滑动窗口切片 | `5-滑动窗口切片.py` |
| 5 | 自适应切片 | `6-自适应切片.py` |
| 6 | 智能自适应切片 | 仅在本对比脚本中（见下） |

**额外策略 — 智能自适应切片：** `smart_adaptive_chunking(text, target_size, min_size, max_size)`  
在句子边界基础上，结合最小/最大长度约束，接近目标长度时根据下一句长度决定是否开新块。

**输出内容：** 每种策略的切片数量、平均长度、长度方差、各块预览，以及汇总对比表。

**适用场景：** 快速评估同一文档在不同策略下的切分效果，辅助选型。

> **说明：** `4-层次切片.py` 未纳入对比测试（需带标题结构的专用文本）；对比测试中的「智能自适应」为对比脚本独有。

## 策略选型参考

| 策略 | 适用文档 | 是否需要 API | 块长度均匀度 | 语义完整性 |
|------|----------|--------------|--------------|------------|
| 固定长度 | 通用纯文本 | 否 | 高 | 中 |
| 句子边界 | 叙述性文本 | 否 | 中 | 较高 |
| LLM 语义 | 主题复杂、结构松散 | 是 | 中 | 高 |
| 层次切片 | Markdown / 带章节标题 | 否 | 中 | 高（结构感知） |
| 滑动窗口 | 需高召回、可接受冗余 | 否 | 高 | 中（有重叠） |
| 自适应 | 多段落说明文 | 否 | 中 | 较高 |

## 技术选型

### 规则切片（脚本 1、2、4、5、6）

- 纯 Python 实现，无外部 NLP 依赖
- 脚本 2、对比测试使用 `re` 模块分句
- 脚本 4 基于行级标题标记检测

### LLM 语义切片（脚本 3、对比测试）

- **OpenAI 兼容客户端** 调用 DashScope：`base_url=https://dashscope.aliyuncs.com/compatible-mode/v1`
- Prompt 要求返回 `{"chunks": [...]}` JSON
- 自动清理 Markdown 代码块包裹（` ```json `）
- LLM 失败时，对比测试脚本会回退到句子边界切片

```powershell
.\.venv\Scripts\Activate.ps1

```

|------|------|------|
| openai | 2.14.0 | 通过兼容接口调用 DashScope LLM（脚本 3、对比测试） |

```powershell

```

### 3. 模型说明

| 脚本 | 默认模型 | 备注 |
|------|----------|------|
| 3-LLM语义切片.py | qwen-turbo-latest | 部分账号可能 403，建议改为 qwen-plus |
| 切片策略对比测试.py | qwen-plus | 已使用较稳定模型 |

```powershell

# 单独运行各策略
python 1-固定长度切片.py
python 2-句子边界切片.py
python 3-LLM语义切片.py      # 需要 DASHSCOPE_API_KEY
```

### 使用自己的文本

将各脚本末尾的 `text = """..."""` 替换为待切分内容，或封装为函数在其他模块中调用：

```python
from importlib import import_module
# 示例：调用固定长度切片
mod = import_module("1-固定长度切片")  # 需注意模块名与文件名
chunks = mod.improved_fixed_length_chunking(your_text, chunk_size=500, overlap=50)
```

更推荐直接 `from` 对应脚本中的函数名导入使用。

## 常见问题

### 1. 脚本 3 报错或未返回结果

- 确认已设置 `DASHSCOPE_API_KEY`
- 将模型名改为 `qwen-plus`（与对比测试脚本一致）
- 检查 LLM 返回是否为合法 JSON；脚本已尝试剥离 ` ``` ` 代码块

### 2. 对比测试中 LLM 切片失败

对比脚本会在异常时**自动回退**到句子边界切片（`semantic_chunking`），终端会打印 `LLM切片失败` 提示。

### 3. 层次切片效果不明显

测试文本需包含 `#`、`##`、`一、`、`1. ` 等标记；纯段落文本请用脚本 2 或 6。

### 4. 滑动窗口切片数量过多

增大 `step_size`（接近 `window_size`）可减少重叠与块数量；`step_size = window_size` 时块之间无重叠。

### 5. chunk_size 单位

本案例均以**字符数**计（中文、英文均按 1 字符），非 token 数。对接 Embedding 模型时需结合模型的最大 token 限制换算。
